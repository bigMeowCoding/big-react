# 核心设计概念

本文说明 Fiber、双缓冲、副作用标记与组件类型。文档索引见 **[AGENTS.md](../AGENTS.md)**。

## Fiber 节点（`FiberNode`）

每个 Fiber 表示一个工作单元，主要字段包括：

| 类别 | 含义 |
|------|------|
| 标识 | `tag`（函数组件 / HostRoot / Host 节点 / 文本等）、`type`、`key` |
| 输入输出 | `pendingProps` → `memoizedProps`；`memoizedState`（Hooks 链表头） |
| 树结构 | `child`、`sibling`、`return`、`index` |
| 副作用 | `flags`（本节点）、`subtreeFlags`（子树冒泡汇总）、`deletions` |
| 双缓冲 | `alternate` 指向另一棵树上的对应节点 |

**`FiberRootNode`**：根对象，持有真实容器（DOM 容器节点）、`current`（当前屏上的 HostRoot Fiber）、`finishedWork`（本次 render 完成后的树）等。

## 双缓冲（current / workInProgress）

- **`root.current`**：当前已提交到宿主的一侧树。
- **工作循环**从 `createWorkInProgress(root.current, …)` 得到 **`workInProgress`**，在树上推进 `beginWork` / `completeWork`，构建 alternate 树。
- **Commit 结束后** `root.current = finishedWork`，完成指针切换。

## 副作用标记（`fiberFlag.js`）

当前实现中与 DOM 变更相关的包括：

- **Placement**：新增或移动节点，需要在 commit 阶段插入到正确父节点与兄弟位置。
- **Update**：例如文本内容变化（在 `completeWork` 里对 HostText 比对并打标）。
- **ChildDeletion**：子树删除，在 commit 阶段卸载并 `removeChild`。

`MutationMask` 聚合了需要在 **mutation 阶段**处理的标记。

## 组件类型（`workTags.js`）

包含 `FunctionComponent`、`HostRoot`、`HostComponent`、`HostText` 等；`beginWork` 按 `tag` 分派到不同更新逻辑。
