# 运行链路

本文说明从 `createRoot().render()` 到 DOM 更新的完整路径，以及各阶段与源码文件的对应关系。文档索引见 **[AGENTS.md](../AGENTS.md)**。

## 总览

从应用侧一次更新到屏幕，可概括为：

1. **创建更新**：根节点或 Hooks 将更新写入 `updateQueue`，并调用 `scheduleUpdateOnFiber`。
2. **调度**：当前实现为 **`ensureRootIsScheduled` → 同步 `performSyncWorkOnRoot`**（无时间切片、无优先级队列）。
3. **Render 阶段**：`workLoop` → 反复 `performUnitOfWork` → `beginWork`（向下）与 `completeWork`（向上冒泡 flags）。
4. **Commit 阶段**：`commitRoot` → `commitMutationEffects`（DOM 插入/更新/删除）。

```mermaid
flowchart TD
  A[createRoot(container).render(element)] --> B[updateContainer]
  B --> C[enqueueUpdate HostRoot.updateQueue]
  C --> D[scheduleUpdateOnFiber]
  D --> E[performSyncWorkOnRoot]
  E --> F[prepareFreshStack: createWorkInProgress]
  F --> G[workLoop: performUnitOfWork]
  G --> H[beginWork]
  H --> I[reconcileChildren / renderWithHooks]
  G --> J[completeWork + bubbleProperties]
  E --> K[commitRoot]
  K --> L[commitMutationEffects]
  L --> M[root.current = finishedWork]
```

## 分阶段说明（与源码对应）

### 入口：`react-dom`（`root.js`）

- **`createRoot(container)`** → `createContainer(container)`（`fiberReconciler.js`）：创建 `FiberRootNode` 与 **HostRoot** Fiber，并初始化根上的 `updateQueue`。
- **`render(element)`**：`initEvent`（如点击委托）后 **`updateContainer(element, root)`**。

### 根更新：`updateContainer`（`fiberReconciler.js`）

- `createUpdate(element)` + `enqueueUpdate(hostRootFiber.updateQueue, update)`。
- **`scheduleUpdateOnFiber(hostRootFiber)`** → 标记路径到根 → **`performSyncWorkOnRoot`**。

### Render 阶段：`workLoop.js`

- **`prepareFreshStack`**：以 `root.current` 为基准创建整棵树的 WIP 根。
- **`workLoop`**：`while (workInProgress !== null)` 执行 **`performUnitOfWork`**。
- **`performUnitOfWork`**：
  - 调用 **`beginWork(fiber)`**，得到下一个子 Fiber 或 `null`；
  - 将 `pendingProps` 同步到 `memoizedProps`；
  - 若有子节点则继续向下；否则进入 **`completeUnitOfWork`**（回溯兄弟与父节点，并 **`completeWork`**）。

### `beginWork`（`beginWork.js`）

- **HostRoot**：`processUpdateQueue` 得到新的子元素（ReactElement），再 **`reconcileChildren`**。
- **HostComponent**：从 `pendingProps.children` 取子节点并 reconcile。
- **FunctionComponent**：**`renderWithHooks`** 执行函数组件，得到子树，再 reconcile。
- **HostText**：无子节点，返回 `null`，直接进入 complete 路径。

**子节点协调**（`childFiber.js`）：根据是否存在 `current.child` 选择 **mount** 或 **reconcile** 路径；对数组子节点处理 key、移动与 **Placement**；删除时收集 **ChildDeletion**。

### `completeWork`（`completeWork.js`）

- **Host 节点**：首次挂载时 **`createInstance` / `createTextInstance`**，**`appendAllChildren`** 把子 DOM 挂到父实例；并 **`bubbleProperties`** 向上聚合 `subtreeFlags`。
- **HostText 更新**：文本变化时打 **Update** 标记（属性更新的精细化与 commit 中的属性变更仍可按教程继续扩展）。

### Commit 阶段：`commitRoot` + `commitWork.js`

- 若根或子树存在 **MutationMask** 相关标记，则 **`commitMutationEffects(finishedWork)`** 遍历 effect 链。
- 按 Fiber 的 **Placement / Update / ChildDeletion** 调用 host 层：**插入容器、文本更新、删除子树** 等。
- 最后 **`root.current = finishedWork`**，提交结束。

Hooks 与宿主层说明见 [05-Hooks与宿主](./05-Hooks与宿主.md)。
